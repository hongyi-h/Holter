## 各 Loss 分量分析

### ✅ 健康学习的组件

1. **beat_mae**: 12.3 → 5.0 (train), 13.1 → 5.8 (val)
   - 持续下降，train/val gap 小（~0.8），无过拟合
   - Beat tokenizer 学到了 P-QRS-T 形态重建

2. **ep_cpc**: 7.1 → 2.2 (train), 6.3 → 2.1 (val)
   - 从 random baseline (~7.1) 大幅下降
   - Val 甚至略低于 train，说明泛化良好
   - Episode encoder 学到了时序预测能力

3. **ep_align**: 7.2 → 3.3 (train), 6.2 → 3.3 (val)
   - 波形和节律 episode token 在对齐
   - Train/val 几乎一致，泛化好

4. **rhythm_mask**: 7.3 → 0.13 (train), 6.5 → 0.10 (val)
   - 最快收敛的 loss，epoch 10 后基本饱和
   - Rhythm branch 完全学会了 VQ code 和 RR 间期预测

5. **report**: 0.71 → 0.30 (train), 0.46 → 0.38 (val)
   - 弱监督信号在工作，但 train/val gap 较大
   - 19 个概念的多标签 BCE，0.38 是合理水平

6. **day_mask**: 0.076 → 0.023 (train), val 始终 0.000
   - Train 上持续下降，Day encoder 在学习 masked episode 重建
   - Val = 0 是因为 eval 模式不生成 mask（预期行为）

7. **vq_loss**: 0.02 → 0.002
   - VQ codebook 收敛良好

### ⚠️ 需要注意

1. **ep_order**: 始终 ~0.68-0.70，接近 random (ln2=0.693)
   - 这个任务没有学到东西，但权重很低 (0.1 × episode weight)
   - 可能原因：相邻 episode 的 CLS token 太相似，交换后差异不够
   - 不影响整体训练，可在消融实验中移除

2. **day_stats**: 仍在 log 中记录但不参与 loss（已从 day loss 移除）
   - 值在 150-350 之间波动，确认移除是正确决策

3. **收敛速度**: epoch 30 后 val loss 下降变慢（2.72 → 2.64，8 epochs 降 0.08）
   - 模型接近当前数据量的容量上限
   - 继续训练可能有微小收益但不值得
